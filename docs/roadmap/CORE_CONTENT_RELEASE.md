# CP28 — Core Content Release

CP28 replaces the one-word bundled catalog with a versioned 100-entry pilot pack while preserving the immutable core, user entry, override, and metadata layers.

Acceptance requires:

- 100 unique core entries;
- zero schema failures;
- zero semantic failures;
- checksums matching the manifest;
- only declared editorial completeness warnings;
- search and detail rendering for manual sample words;
- Library visibility and layer filtering;
- persistence of existing user overrides and study metadata;
- production bundle budget remaining green.
