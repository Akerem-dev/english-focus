# Test Strategy

- Unit: normalizers, validators, migrations, presenters, instruction builders.
- Integration: use cases with fakes and SQLite adapters.
- Component: screen states and overlays.
- Accessibility: keyboard, focus, semantics, contrast-related structure.
- E2E: missing-word flow, import, duplicate resolution, library management, backup restore.
- Performance: 10,000-entry search and large pack import.
